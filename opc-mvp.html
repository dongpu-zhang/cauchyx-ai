Core Flow — MVP User Journey

1
Creator logs in to the Web Dashboard
Single-page app authenticates via Google OAuth or email/password. Session is managed with a JWT stored in an httpOnly cookie. Creator sees their project list.
2
Creator submits a Story Input
Free-form natural language goal — e.g. "Make a 10-minute YouTube explainer on AI agents for non-technical founders." — sent to the backend via a REST POST. A new project record is created in PostgreSQL.
3
Planner Agent generates a structured plan
Claude Haiku reads the story, retrieves relevant context from ChromaDB (past topics, creator style profile), and outputs a structured JSON plan: episode outline, target audience, key talking points, content hooks.
4
Creator Agent produces the full content package
Claude Sonnet takes the plan and writes: full script with timestamps, thumbnail text brief, YouTube description (SEO-optimised), and 10 hashtags. Output is stored as structured JSON in PostgreSQL.
5
Critic Agent scores and approves or requests revision
Claude Haiku evaluates against rubric: clarity, hook strength, factual accuracy, tone match, SEO quality. Returns a score (1–10) plus specific revision notes. If score < 7, a revision request loops back to the Planner with structured feedback. Max 3 revision cycles.
6
Creator reviews and approves the final output
Dashboard displays the generated package with Critic score and inline suggestions. Creator can edit any section directly, accept as-is, or request a manual re-run. A approved_at timestamp is set on the record.
7
One-click publish to YouTube (or copy to clipboard)
Via YouTube Data API v3 the description, tags, and thumbnail brief are uploaded as a draft. The script is bundled as a downloadable .pdf teleprompter. Memory layer is updated with the completed project for future context recall.
04

What This Validates

H1 · Core Value
Creators will pay for agent-assisted content production
If 10+ paying users run the loop weekly, the core value proposition is real. Metric: weekly active projects per user.
H2 · Quality Bar
3-agent loop produces publish-ready content without human editing
If Critic score averages >7.5 on first approval and users publish >60% of outputs without edits, quality is validated.
H3 · Unit Economics
Claude API cost stays below $2 per content package
If average token cost per full run (Planner + Creator + Critic) stays under $2, a $29/mo subscription plan is viable with >40% gross margin.
H4 · Memory Lift
Persistent memory improves quality over time
If Critic scores on run #5+ for a project are measurably higher than run #1 (style learning), the ChromaDB memory layer earns its architectural cost.

Build Timeline — 6-Week Plan

WK 1
Foundation & Auth
  • Provision cloud VM (DigitalOcean / Hetzner), configure Docker Compose
  • PostgreSQL schema: users, projects, content_packages
  • FastAPI backend skeleton with JWT auth + Google OAuth
  • React/Next.js frontend scaffold — login, dashboard shell
WK 2
Planner Agent + Memory Layer
  • Claude API integration (Haiku) with structured JSON output via tool use
  • ChromaDB setup: creator style profile + project context collections
  • Story Input UI — text area, project name, target platform selector
  • Plan display view — outline, talking points, estimated length
WK 3
Creator Agent + Full Content Package
  • Claude Sonnet integration — long-context script generation
  • Output schema: script sections, thumbnail brief, description, tags
  • Content review UI — tabbed view per output section, inline edit
  • PDF export of script for teleprompter use
WK 4
Critic Agent + Revision Loop
  • Critic prompt — scoring rubric, revision note format
  • Revision loop orchestration — max 3 cycles, escalation to user
  • Quality log written to ChromaDB after each run
  • Score display + revision diff view in dashboard
WK 5
Stripe Billing + YouTube Publish
  • Stripe Checkout — single $29/mo plan, webhook for activation
  • Usage gate — trial: 3 free runs, then requires active subscription
  • YouTube Data API v3 — OAuth connect, upload draft description + tags
  • Publish status tracking in dashboard — draft / published / failed
WK 6
Polish, Security & Launch
  • Rate limiting, input validation, error boundary UI
  • HTTPS via Caddy reverse proxy, secrets in env vault
  • End-to-end smoke tests for full story → publish flow
  • Soft launch to 10 beta creators — feedback collection via Tally form
