一、场景概述
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

供给方（专家）
  · 身份：从事餐饮行业环保合规咨询 15 年的独立顾问
  · 专长：EPA 联邦法规 / 州级卫生标准 / 市政环保条例 / 食品安全法规
  · 痛点：过去靠人工追踪法规网站 + 手写邮件通知客户，每月耗费 80+ 小时

需求方（客户）
  · 身份：美国中小餐馆经营者（独立餐馆、连锁小店）
  · 痛点：看不懂政府法规原文、不知道什么时候会被罚、没钱聘合规律师
  · 愿意付费：$5/月（相当于一杯咖啡），换取"不被突击检查罚款"的安心感

商业模式
  ┌─────────────────────────────────────────────────────────┐
  │  餐馆 → 每月支付 $5 订阅费 → 专家账户                    │
  │  OPC 平台 → 从每笔订阅中收取 20% 服务费 = $1             │
  │  专家净收入 → $4 / 餐馆 / 月                             │

阶段 1：每周自动监控循环（Harness OPC 工作循环）
  ┌──────────────────────────────────────────────────────────────────────────┐
  │                                                                          │
  │  [Think]          [Do]             [Check]          [Improve]            │
  │  任务规划      →  执行扫描      →  内部评审      →  记忆更新             │
  │  Planner Agent    RegScanner       Critic Agent      Memory Agent        │
  │                   LegalInterp.                                           │
  │                   RiskMapper                                             │
  │                   AlertComposer                                          │
  │                   Dispatcher                                             │
  └──────────────────────────────────────────────────────────────────────────┘

  Think（Planner Agent）
    · 加载上周记忆：已推送法规清单、各餐馆合规状态
    · 确定本周扫描范围：哪些来源、哪些关键词
    · 输出：本周扫描任务清单

  Do（4 个执行 Agent 串行）
    · RegScanner Agent：抓取并筛选
    · LegalInterpreter Agent：解读原文
    · RiskMapper Agent：映射到各餐馆
    · AlertComposer Agent：生成个性化预警
    · Dispatcher Agent：发送给受影响餐馆

  Check（Critic Agent）
    · 评估本周预警的质量和完整性
    · 确认每条预警至少包含：违规后果金额 + 整改截止日期 + 3 步行动建议
    · 质量分 < 8 → 打回 AlertComposer 重写

  Improve（Memory Agent）
    · 将本周发现的新法规写入向量库
    · 记录哪些餐馆已收到提醒
    · 更新各法规的生效日期倒计时

三、集成的 6 个 Agent 及职责
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─────────────────────┬───────────────────────────────────────────────────────┐
│ Agent 名称           │ 职责与工具                                             │
├─────────────────────┼───────────────────────────────────────────────────────┤
│ 1. RegScanner       │ 职责：自动抓取政府网站，识别新增/变更法规               │
│    （法规扫描员）    │ 工具：Playwright 无头浏览器 + RSS 订阅 + PDF 解析      │
│                     │ 来源：EPA.gov / FDA.gov / 州 DEQ / 市政网站            │
│                     │ 输出：新法规列表（标题 + 来源 URL + 发布日期 + 原文）  │
│                     │ 触发：每周一 06:00 UTC Cron 触发                       │
├─────────────────────┼───────────────────────────────────────────────────────┤
│ 2. LegalInterpreter │ 职责：将法律原文翻译为餐馆经营者能看懂的语言            │
│    （法规解读员）    │ 工具：Claude API（专家微调模型）+ 法规知识库            │
│                     │ 输出：结构化解读                                        │
│                     │   · 法规摘要（3 句话）                                 │
│                     │   · 生效日期 + 执行机构                                │
│                     │   · 违规后果（罚款金额 / 吊销执照风险）                │
│                     │   · 受影响的餐馆类型                                   │
├─────────────────────┼───────────────────────────────────────────────────────┤
│ 3. RiskMapper       │ 职责：将法规映射到具体受影响的餐馆                      │
│    （风险映射员）    │ 工具：PostgreSQL（餐馆档案查询）+ 规则引擎              │
│                     │ 逻辑：法规类型 × 餐馆属性 → 匹配受影响名单             │
│                     │   例：室外座位 > 200 sqft 的法规                       │
│                     │       → 查询有室外座位属性的餐馆 → 触发预警            │
│                     │ 输出：受影响餐馆 ID 列表 + 每家的风险等级（高/中/低）  │
├─────────────────────┼───────────────────────────────────────────────────────┤
│ 4. AlertComposer    │ 职责：为每家餐馆生成个性化预警邮件                      │
│    （预警撰写员）    │ 工具：Claude API + 餐馆档案 + 预警模板库               │
│                     │ 个性化维度：餐馆名称 / 所在市 / 具体违规风险点 /        │
│                     │            距生效日期剩余天数                          │
│                     │ 输出模板（每封邮件必含）：                              │
│                     │   [!] 紧急程度标签（HIGH/MEDIUM/LOW）                  │
│                     │   法规名称 + 一句话摘要                                │
│                     │   "您的餐馆受影响，因为：[具体原因]"                  │
│                     │   违规后果：最高罚款 $[X] 或 [后果]                   │
│                     │   距生效日期：[N] 天                                   │
│                     │   三步整改建议：Step 1 / Step 2 / Step 3              │
│                     │   "有问题？直接回复此邮件咨询专家"                    │
├─────────────────────┼───────────────────────────────────────────────────────┤
│ 5. Dispatcher       │ 职责：多渠道推送预警给受影响餐馆                        │
│    （分发员）        │ 工具：Resend（邮件 API）+ Twilio（SMS）                │
│                     │ 逻辑：高风险 → Email + SMS；中风险 → Email；           │
│                     │       低风险 → Email（周报合并）                       │
│                     │ 记录：发送时间戳 + 已读回执 → 写入 PostgreSQL          │
│                     │ 退订处理：一键退订链接 → 自动更新订阅状态              │
├─────────────────────┼───────────────────────────────────────────────────────┤
│ 6. QA Agent         │ 职责：24/7 回答餐馆主的跟进问题                        │
│    （问答客服）      │ 工具：Claude API + 专家知识库 + 该餐馆历史预警记录     │
│                     │ 触发：餐馆主回复预警邮件 / 访问客户门户聊天            │
│                     │ 边界：涉及法律建议时，附加"建议咨询持牌律师"免责声明  │
│                     │ 升级：QA Agent 无法回答 → 转发给专家 + 24h 内回复      │
└─────────────────────┴───────────────────────────────────────────────────────┘

四、OPC 平台底层支持模块
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OPC 平台为专家透明提供以下基础能力，专家无需自己开发：

1. 任务调度（Webhook + Cron）
   · 每周一定时触发 RegScanner
   · 法规紧急变更时支持手动触发
   · 任务队列管理（避免同时处理时资源冲突）

2. 多租户订阅管理
   · 每家餐馆有独立的数据沙箱（档案 / 预警历史 / 问答记录）
   · 订阅状态实时同步（付费 → 激活；逾期 → 暂停服务）
   · 批量导入餐馆档案（CSV 上传）

3. 计费与分润系统（Stripe 集成）
   · 餐馆通过专家专属链接订阅，自动扣款 $5/月
   · OPC 平台自动扣留 20%（$1）
   · 专家账户自动到账 $4/餐馆/月
   · 每月结算报告（收入明细 / OPC 服务费发票）

4. 知识库与长期记忆（向量库）
   · 所有已解读法规 embedding 存储（ChromaDB）
   · 法规变更追踪（版本对比，新旧条款 diff）
   · 餐馆合规状态时间线（每家餐馆何时收到哪条预警）
   · 专家知识库（上传 PDF → 自动 embedding → Agent 可检索）

5. FTaaS（专属模型微调）
   · 专家上传历史解读文档 → 微调 LegalInterpreter 专属模型
   · 微调后：法规解读准确率提升，输出风格与专家一致
   · 每季度自动增量微调（新增法规样本）

6. 专家管理控制台
   · 法规扫描结果预览（发布前审核）
   · 预警发送前 24h 审核队列（专家可编辑/驳回）
   · 订阅者列表 + 每家餐馆合规状态看板
   · 月度收入 + 服务用量报告

7. 客户门户（餐馆视角）
   · 餐馆主登录，查看历史预警
   · 在线咨询 QA Agent
   · 合规状态自查清单（勾选已完成的整改项）
   · 下载预警报告（供餐馆留存备查）

8. 安全与合规
   · 数据隔离：每家餐馆数据不互通
   · 审计日志：每条预警的生成时间 + 发送时间 + 已读状态
   · 免责条款：自动附加在每封预警邮件底部

06:00 UTC  OPC Cron 触发，启动本周 RegWatch 工作循环

── [Think] Planner Agent ──────────────────────────────────────────────────────

  Planner 从记忆库加载：
    · 上周推送：3 条法规预警（德州废水排放修订、加州噪音限制）
    · 本周重点：加州 CalRecycle Q3 新增条款已进入 90 天窗口期

  制定本周任务：
    Task 1  扫描 EPA.gov（关键词：restaurant, grease, stormwater）
    Task 2  扫描 CalRecycle.ca.gov（关键词：organic waste, food service）
    Task 3  扫描 CDPH.ca.gov（加州卫生部）
    Task 4  扫描 TCEQ.texas.gov（德州环境质量委员会）

── [Do] 执行阶段 ──────────────────────────────────────────────────────────────

06:05  RegScanner Agent 启动，并行抓取 4 个来源

  [发现 1] EPA.gov
    标题：Final Rule - Supplemental Effluent Limitations for Food Service
           Facilities (40 CFR Part 453)
    发布日期：2026-06-05
    生效日期：2026-09-05（91 天后）
    状态：NEW（上周未见）
    原文摘要：餐馆室外排水必须加装二级油脂截留装置，
             罚款：每违规天 $1,000–$10,000

  [发现 2] CalRecycle.ca.gov
    标题：SB 1383 Enforcement Update - Restaurant Organic Waste Compliance
    发布日期：2026-06-06
    生效日期：已生效（补充执行细则）
    状态：UPDATED（已有法规的执行细则更新）
    原文摘要：2026 年 Q3 起，县级执法人员开始突击检查有机废物分类，
             首次违规警告，第二次违规 $200–$500/天

06:18  LegalInterpreter Agent 处理两条发现

  [解读 1] EPA 新规
    摘要（3 句话）：
      EPA 要求所有有室外排水的餐饮设施，在 91 天内加装二级油脂截留器。
      未合规者每天最高被罚 $10,000，执法机构为地方水务局。
      这是对现行法规的强化，不是新法规，但增加了室外排水的具体要求。
    受影响类型：有室外座位 + 有室外洗涤区 + 有停车场清洗区的餐馆
    整改成本估算：油脂截留器安装 $800–$3,000（一次性）

  [解读 2] CalRecycle SB 1383 更新
    摘要（3 句话）：
      加州将开始实际执法 SB 1383 有机废物分类规定，餐馆必须分开收集厨余。
      2026 年 Q3 起（7 月 1 日）县级检查员开始上门突击检查。
      首次违规仅警告，但第二次起每天罚款 $200–$500，连续计算。
    受影响类型：加州境内所有餐馆（无差别）
