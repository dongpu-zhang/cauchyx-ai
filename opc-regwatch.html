Agent Details

6 个 Agent 职责详情

① Planner Agent
Think · 任务规划
读取上周记忆，制定本周 4 个扫描任务清单，分配优先级
ChromaDB 读取Claude API
② RegScanner Agent
Do · 法规抓取
Playwright 无头浏览器 + RSS 订阅 + PDF 解析，并行扫描 EPA / FDA / 州 DEQ / 市政
PlaywrightPDF ParserRSS
③ LegalInterpreter Agent
Do · 法规解读
调用专家微调模型，将法律原文转化为 3 句话摘要 + 罚款金额 + 受影响类型
Fine-tuned ClaudeFTaaS
④ RiskMapper Agent
Do · 风险映射
SQL 查询餐馆档案，按属性匹配（室外座位/所在州/营业类型），输出受影响名单 + 风险等级
PostgreSQL规则引擎
⑤ AlertComposer Agent
Do · 预警撰写
为每家餐馆生成含店名、具体风险点、3 步整改建议的个性化预警邮件，支持中英双语
Claude API模板库个性化
⑥ Critic Agent
Check · 质量审核
评分 0–10，验证每封预警是否包含：违规后果金额 + 截止日期 + 3 步建议。<7 分打回重写
质量门控最多 3 轮
⑦ Dispatcher Agent
Do · 多渠道发送
高风险 → Email + SMS；中风险 → Email；低风险 → 周报合并。记录发送状态到 PostgreSQL
ResendTwilio99% 成功率
⑧ Memory Agent
Improve · 记忆更新
将新法规写入向量库，更新各餐馆合规状态时间线，供下周 Planner 读取
ChromaDB持续学习
⑨ QA Agent（常驻）
24/7 · 餐馆问答
回答餐馆跟进问题，引用历史预警和知识库，复杂问题转发给专家，24h 内回复
