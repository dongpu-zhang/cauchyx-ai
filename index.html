Domain Ontology — Semantic Constraint Layer

A formal OWL 2 DL ontology translates PINN outputs into semantically grounded business assertions. Figure 1 | Lang-PINN 系统架构总览。 (A) 核心推理流程：用户自然语言经 PDE Agent 解析为符号偏微分方程（含周期性、几何与多尺度特征）， PINN Agent 从候选架构库中选取最优网络结构，Code Agent 生成模块化求解代码， Feedback Agent 执行闭环评估并将误差与收敛性能数据反馈至 PDE Agent 与 PINN Agent，驱动迭代优化。 (B) 候选架构库：涵盖 MLP、KAN、Transformer、CNN 四类通用架构，以及 CauchyNet 与 XNet 两类原创架构（粗框标注）； PINN Agent 通过虚线选择通道管理架构库，选定架构经实线通道注入 Code Agent。 (C) 底层数学基础：复分析中的柯西积分公式为 CauchyNet 与 XNet 提供理论支撑，实现对全纯函数在任意点的精确解析表达， 在近似精度上超越基于局部多项式的标准 PINN 方法。 实线箭头 = 数据 / 代码流；虚线箭头 = 选择 / 引用关系；粗边框节点 = 本工作原创贡献。
