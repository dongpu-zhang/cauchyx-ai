一、MVP 定位
────────────────────────────────────────────────────────────────────────────────

产品名称：OPC Lite
目标用户：独立内容创作者（YouTube / Newsletter 方向）
核心价值：一句话描述目标（Story），AI 自动完成内容策划 → 创作 → 审核 → 发布
验证目标：证明"三 Agent 流水线"可持续产出可发布质量内容，并找到愿意付费的用户


二、MVP vs 完整 OPC 平台对比
────────────────────────────────────────────────────────────────────────────────

维度                  完整 OPC 平台                   MVP (OPC Lite)
──────────────────────────────────────────────────────────────────────────────
服务赛道              5 个垂直（内容/工具/电商/教育/企业）  1 个：内容创作者 OPC
Agent 数量            20–50 个虚拟 Agent               3 个：Planner / Creator / Critic
用户入口              Web + 微信 + Telegram + App + API  Web 控制台（仅）
LLM 策略              开源自部署 + 闭源路由 + FTaaS       Claude API 单一接入
记忆系统              向量库 + 长期记忆 + 行业 SOP         ChromaDB（单用户项目上下文）
平台能力              7 大模块（计费/安全/合规/分析…）      登录 + 项目列表 + Stripe 订阅
外部集成              20+ 个（社交/支付/办公/通讯）         YouTube Data API（1 个）
基础设施              OCI 多节点 + 国内 GPU 池            单台云 VM + PostgreSQL
合规层                ISO 27001 / SOC 2 / GDPR          基础日志记录
计费方案              多币种 / 发票 / 对账                 单计划 $29/月（Stripe）
团队协作              多团队 / 权限 / SSO                 单用户
模型管理              Prompt 版本 / A-B 测试              固定 Prompt 模板
数据分析              BI 看板 / 监控告警                   项目完成率统计


三、MVP 架构（三层结构）
────────────────────────────────────────────────────────────────────────────────

层级 1：用户入口
  └── Web 控制台
        · 登录（邮箱 + 密码，JWT）
        · 新建 / 查看 Story（自然语言目标）
        · 任务进度看板（实时状态）
        · 内容预览 + 一键发布按钮

层级 2：Agent Pipeline（核心引擎）
  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  [Planner Agent]  ──→  [Creator Agent]  ──→  [Critic Agent]
  │       │                                           │     │
  │       └───────────── 反馈循环（评分 < 7）──────────┘     │
  │                                                         │
  │              [Shared Memory — ChromaDB]                 │
  │         风格偏好 | 项目上下文 | 质量日志                   │
  └─────────────────────────────────────────────────────────┘

  Planner Agent
    · 职责：解析 Story，制定 2 周内容冲刺计划
    · 输出：结构化任务列表（主题 / 关键词 / 发布日期）
    · 工具：Claude API + ChromaDB 读取（历史风格）

  Creator Agent
    · 职责：根据任务生成完整内容包
    · 输出：视频脚本 / 缩略图文案 / 视频描述 / Tags 列表
    · 工具：Claude API + ChromaDB 读取（项目上下文）

  Critic Agent
    · 职责：对内容包进行质量评分（0–10 分）
    · 评分维度：相关性 / 可读性 / SEO 友好性 / 品牌一致性
    · 阈值：>= 8 分 → 通过；< 7 分 → 触发修订请求回 Planner
    · 最大循环：3 轮（避免死循环）

层级 3：基础设施
  Claude API      ← LLM 骨干（所有 Agent 共用）
  PostgreSQL      ← 项目数据、用户数据、任务状态
  ChromaDB        ← 向量记忆（风格 embedding / 质量日志）
  Stripe          ← 订阅计费（$29/月单一计划）
  YouTube API     ← 内容发布（OAuth 2.0 授权）
  云 VM（单台）    ← 部署（Render / Railway / DigitalOcean）


四、核心用户流程（7 步）
────────────────────────────────────────────────────────────────────────────────

Step 1  用户登录 Web 控制台，点击「新建 Story」
        输入自然语言目标，例如：
        "帮我在 3 个月内将 YouTube 频道增长到 1 万订阅，聚焦 AI 工具教程"

Step 2  Planner Agent 接收 Story
        · 从 ChromaDB 加载用户风格偏好（如果已有历史）
        · 生成 2 周冲刺计划：8 个视频主题，含关键词和优先级
        · 写入 PostgreSQL（任务状态：pending）

Step 3  Creator Agent 逐任务执行
        · 按优先级取第一个任务
        · 调用 Claude API 生成内容包：
          - 视频脚本（含片头钩子、正文结构、CTA）
          - 缩略图文案（3 个备选）
          - 视频描述（含时间戳、关键词布局）
          - Tags 列表（15–20 个）

Step 4  Critic Agent 质量评审
        · 对内容包打分（0–10）
        · 若分数 >= 8：输出「APPROVED」
        · 若分数 < 7：输出修订说明，发回 Planner Agent
        · 最多重复 3 轮

Step 5  内容包通过
        · 写入 ChromaDB：更新风格记忆 + 质量日志
        · PostgreSQL 任务状态更新为「ready」
        · 前端通知用户「内容已准备好，等待确认」

Step 6  用户在 Web 控制台预览内容
        · 可手动微调文案（可选）
        · 点击「确认发布」

Step 7  发布
        · 调用 YouTube Data API 上传/安排发布
        · 记录发布时间戳到 PostgreSQL
        · ChromaDB 写入发布反馈标记（供下次 Planner 参考）


五、4 个业务假设验证目标
────────────────────────────────────────────────────────────────────────────────

H1  价值假设
    假设：内容创作者愿意为 AI 内容助手付费
    成功指标：前 3 个月获得 5 个付费用户（$29/月）
    验证方式：冷启动邀测 20 人，转化率 > 25%

H2  质量假设
    假设：3-Agent 流水线可产出可直接发布质量的内容
    成功指标：Critic 评分 >= 8，用户手动修改 < 2 处
    验证方式：前 50 次运行记录人工修改次数

H3  单位经济假设
    假设：单次运行成本可控，毛利可持续
    成功指标：单次 LLM 成本 < $1.50，毛利率 > 50%
    验证方式：追踪每次运行的 Claude API token 消耗

H4  记忆增益假设
    假设：风格记忆积累后内容质量提升
    成功指标：第 5 次运行起，Critic 评分比第 1 次提升 >= 1 分
    验证方式：对比同一用户早期 vs 后期 Critic 评分曲线


六、单位经济测算
────────────────────────────────────────────────────────────────────────────────

成本项目                      估算
──────────────────────────────────────────────────────────────────────────────
Claude API（每次运行约 30K tokens）   $1.40 / 运行
基础设施（VM + DB + ChromaDB）        $0.80 / 用户 / 月
（按每用户每月 10 次运行）

定价                          $29 / 用户 / 月

月度成本（10 次运行/用户）     $1.40 × 10 + $0.80 = $14.80
月度毛利                      $29 - $14.80 = $14.20
毛利率                        约 49%（10 次）→ 54%（8 次标准用量）

盈亏平衡点                    10 个付费用户
  （假设固定运营成本 $100/月，即 VM + 域名 + Stripe 手续费）


七、6 周交付计划
────────────────────────────────────────────────────────────────────────────────

Week 1–2：Agent 核心
  · 3-Agent 基础框架（LangGraph 或自定义循环）
  · Claude API 集成（Planner / Creator / Critic Prompt 调试）
  · ChromaDB 初始化（风格 embedding 写入 / 读取）
  · 评分逻辑 + 反馈循环（最大 3 轮）
  · 单元测试：5 个 Story → 内容包生成通过率 > 80%

Week 3：Web 控制台
  · 用户登录（JWT）
  · Story 输入表单
  · 任务进度看板（实时 WebSocket 状态推送）
  · 内容预览页（脚本 / 描述 / Tags）

Week 4：外部集成 + 计费
  · YouTube Data API（OAuth 2.0 + 视频上传 / 发布安排）
  · Stripe 订阅（$29/月计划 + Webhook 处理）
  · 用户订阅状态验证（未付费 → 限流）

Week 5：内测（5 位用户）
  · 邀请 5 位 YouTube 创作者内测
  · 收集反馈：内容质量打分、流程顺滑度、价格接受度
  · 记录每次运行的 API 成本

Week 6：迭代 + 稳定
  · 根据内测反馈调整 Prompt（Critic 评分标准校准）
  · Bug 修复 + 性能优化
  · 准备公开测试（Landing Page + 等待列表）


八、技术栈选型
────────────────────────────────────────────────────────────────────────────────

后端
  · Python 3.11 + FastAPI
  · PostgreSQL 15（项目 / 用户 / 任务数据）
  · ChromaDB（向量记忆）
  · Anthropic SDK（Claude API）
  · Google API Python Client（YouTube）
  · Stripe Python SDK

前端
  · Next.js 14（App Router）
  · TailwindCSS
  · WebSocket（任务状态实时推送）

部署
  · Railway 或 Render（单一云 VM，$25/月起）
  · PostgreSQL（Railway 托管版）
  · 域名 + HTTPS（Let's Encrypt）

Agent 编排
  · 自定义 Python 循环（轻量，不引入 LangChain 依赖）
  · 状态机：PLANNING → CREATING → REVIEWING → APPROVED / REVISION


九、后续扩展路径（MVP 之后）
────────────────────────────────────────────────────────────────────────────────

Phase 2（月 3–4）
  · 新增第 2 个服务垂直：AI 工具 OPC（SaaS 自动化营销）
  · 新增入口：Telegram Bot
  · 开源模型自部署（Qwen / DeepSeek）降低 LLM 成本

  · 多用户 / 团队协作

Phase 4（月 7–12）
  · 完整 OPC 平台（5 个垂直赛道）
  · OCI 节点部署
  · Story → Auto Execution（完整 20-50 Agent Matrix）


十、风险与对策
────────────────────────────────────────────────────────────────────────────────

风险 1：Claude API 成本超预期
  对策：设置每次运行 token 上限（30K），超出截断并提示用户

风险 2：YouTube API 配额限制
  对策：申请生产级配额；MVP 阶段限每用户每天 3 次发布

风险 3：Critic 评分标准不稳定
  对策：前 5 位内测用户人工标注 50 条样本，校准 Prompt

风险 4：用户不愿授权 YouTube OAuth
  对策：提供「下载模式」（内容包 ZIP 下载，用户自行上传）

风险 5：付费转化率低
  对策：前 30 天免费试用 5 次运行；收集 NPS 后调整定价
